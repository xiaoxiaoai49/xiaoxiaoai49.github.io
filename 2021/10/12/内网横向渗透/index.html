hydra

参数详解：
-R 根据上一次进度继续破解
-S 使用SSL协议连接
-s 指定端口    这是老版本的，新版本直接ip 后门跟端口就行了
-l 指定用户名
-L 指定用户名字典(文件)
-p 指定密码破解
-P 指定密码字典(文件)
-e 空密码探测和指定用户密码探测(ns)
-C 用户名可以用:分割(username:password)可以代替-l username -p password
-o 输出文件
-t 指定多线程数量，默认为16个线程
-vV 显示详细过程
server 目标IP
 
service 指定服务名(telnet ftp pop3 mssql mysql ssh ssh2......)

Hydra使用案例

使用hydra破解ssh的密码
hydra -L users.txt -P password.txt -vV -o ssh.log -e ns IP ssh
破解smb：

hydra -l administrator -P pass.txt IP 
smb
破解rdp：
hydra IP rdp -l administrator -P pass.txt -V
破解telnet
 
hydra IP telnet -l 用户 -P 密码字典 -t 32 -s 23 -e ns -f -V
破解ftp：
hydra IP ftp -l 用户名 -P 密码字典 -t 线程(默认16) -vV
 
hydra IP ftp -l 用户名 -P 密码字典 -e ns -vV
破解cisco：
hydra -P pass.txt IP cisco
 
hydra -m cloud -P pass.txt 10.36.16.18 cisco-enable

爆ssh:
hydra -l administrator -P pass.txt -t 30 -vV -e ns telnet://192.168.18.131
-e 检查空密码后面要加一个ns  高亮就是爆出来了

Crunch

crunch 是一种创建密码字典的工具，该字典通常用于暴力破解。
使用crunch工具生成的密码可以发送到终端、文件或另一个程序。

用法：

crunch min max charset options

crunch 6 8 zhang2008 -o /root/pass.txt
crunch 7 7 –f charset.lst numeric -o /root/pass.txt

crunch 8 8 -f /usr/share/crunch/charset.lst numeric
这样生成大约有858M太大了
也可以边生成变爆破，这样没有文件落地

离线密码破解

Hashcat

这有篇文章可以查看一下参数
https://blog.csdn.net/cpongo3/article/details/93995803


Hashcat系列软件是比较牛逼的密码破解软件，系列软件包含
Hashcat、oclHashcat；还有一个单独新出的oclRausscrack。
其区别为Hashcat只支持cpu破解；
oclHashcat和oclGausscrack则支持gpu加速。
oclHashcat则分为AMD版和NIVDA版。

用法：
使用字典破解密码
hashcat a.hash -m 0 -a 0 pass  -o 123.txt   （pass是个密码字典) 
掩码破解
破解7位纯数字密码
hashcat -m 0 -a 3 test.hash ?d?d?d?d?d?d?d
破解7位字母加数字的 xiao2008
hashcat -m 0 -a 3 test.hash ?l?l?l?ld?d?d?d
自定义组合破解
破解5位未知密码并且自定义密码组合
hashcat -m 0 -a 3 test.hash --custom-charset1=xiao --custom-charset2=?d  ?1?2?2?2?2

hashcat --help
他的爆破原理相当于
我有一个密文字典我对应使用加密与密文进行匹配，匹配上就爆出来了
甚至能识别wifi的加密类型
一秒可以加载1.4亿个密码
